# Codingal Release Update

Use this skill when the user asks for a release update, launch update, or short post-release summary for the team.

## Goal

Write a short and sweet non-technical release update about the current PR or recent changes.

## Format

1. Start with `Release Update: [Brief title]`
2. Add a brief greeting announcing the release.
3. Add a `Key Updates:` section.
4. List the main changes as bullets using `—`.
5. End with a short impact statement mentioning that hopefully students or users will love the changes.

## Style Rules

1. Keep it concise and easy to read.
2. Use simple, non-technical language.
3. Focus on benefits and improvements, not implementation details.
4. Keep an excited, positive tone.
5. Use at most 10-12 bullets.
6. Prefer user-facing improvements over internal refactors.

## Working Notes

1. If the request references a PR, use the current branch PR context when available.
2. If the change is mostly technical, translate it into outcome-focused language.
3. Skip low-signal internal details unless they directly affect users or team operations.
